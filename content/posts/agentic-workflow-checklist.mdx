## 为什么很多 Agent 流程止步于 Demo

因为缺少最基础的生产约束：

- 输入边界不清晰
- 执行结果不可验证
- 没有回滚路径

## 一条可靠的交付链路

### 输入校验

在执行前做 schema 校验，并对高风险参数做白名单。

### 可观测性

至少记录三个维度：

1. 请求上下文
2. 执行时长
3. 失败原因分类

### 结果门禁

让 Agent 输出结构化结果，再由规则引擎判断是否放行。

```ts
type AgentResult = {
  summary: string;
  changedFiles: string[];
  testsPassed: boolean;
};

export function canShip(result: AgentResult): boolean {
  return result.changedFiles.length > 0 && result.testsPassed;
}
```

## 发布策略建议

- 先灰度到内部环境。
- 再扩大到低风险业务线。
- 最后接入核心链路。

把这套策略配进平台后，Agent 才是工程资产，不是演示脚本。
